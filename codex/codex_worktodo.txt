Priority 1 - Security and Safety
================================

Task 1: Add .gitignore and protect secrets
------------------------------------------
Goal:
.env, logs, cache, vendor, backups aur generated files ko git se protect karna.

Prompt:
WealthDash PHP/XAMPP project ka security cleanup karo. Project root mein
.gitignore add/update karo. .env, logs/*.log, logs/cache/, vendor/, backups,
database backup files, IDE files, OS files aur temporary generated files ignore
karo. .env.example ko preserve rakho. Existing files ko delete mat karo. Agar
.env tracked hone ka risk dikhe to final response mein secret rotation warning
do. Change minimal aur safe rakho.


Task 2: Audit and fix CSRF exemption list
-----------------------------------------
Goal:
State-changing API actions par CSRF mandatory karna.

Prompt:
api/router.php mein CSRF exemption list audit karo. Sirf pure read-only actions
ko csrfExempt mein rakho. Add, edit, update, delete, save, toggle, import,
mark, clear, flush, reset, trigger, restore, truncate, create, revoke, sync,
fix, scan, merge, dismiss, upload, run jaise state-changing actions ko exemption
se remove karo. Existing behavior ko break na karte hue changes karo. High-risk
actions ki short list final response mein do. Syntax check run karo.


Task 3: Harden admin destructive actions
----------------------------------------
Goal:
Admin DB truncate/restore/cron trigger jaise actions safe banana.

Prompt:
WealthDash admin APIs ka safety hardening karo. api/router.php aur related
api/admin files inspect karo. DB truncate, restore, backup delete, cron trigger,
rate limit flush/reset, user delete/reset-password jaise actions ke liye admin
role check, CSRF check, audit_log call, aur explicit confirmation parameter
require karo. Existing UI/API contract ko minimally adjust karo. Har destructive
action ke liye clear JSON error return karo agar confirmation missing ho.


Task 4: Improve session security headers
----------------------------------------
Goal:
Security headers aur session behavior strengthen karna.

Prompt:
config/config.php aur common layout/API bootstrap mein security headers add
karo: X-Frame-Options, Referrer-Policy, Permissions-Policy, X-Content-Type-
Options, aur production mein basic Content-Security-Policy. Session cookie
settings review karo; httponly, samesite aur secure behavior preserve karo.
Local XAMPP development break nahi hona chahiye. Changes scoped rakho.


Task 5: Add API rate limit for sensitive endpoints
--------------------------------------------------
Goal:
Login ke alawa sensitive APIs par abuse protection.

Prompt:
WealthDash mein sensitive API actions ke liye lightweight rate limiting add
karo. Existing rate limit helpers ya database tables search karo. Agar reusable
helper available ho to use karo, warna minimal helper banao. Apply to AI chat,
admin cron trigger, import endpoints, backup/restore, and auth-sensitive
actions. Response JSON format existing json_response helper ke saath consistent
rakho.
