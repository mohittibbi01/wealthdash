WealthDash Project Analysis - Codex Suggestions and Work
=======================================================

Date: 02 June 2026
Project Path: C:\xampp\htdocs\wealthdash

Short Summary
-------------
WealthDash ek PHP/XAMPP based India-focused personal finance dashboard hai.
Project ka feature scope kaafi strong hai: Mutual Funds, Stocks, FD, NPS,
Crypto, Tax, Goals, Reports, Admin, Cron jobs, 2FA, AI advisor aur integrations
jaise modules already present hain.

Project abhi "feature-rich but maintenance-heavy" stage par lag raha hai.
Matlab features bahut hain, lekin security audit, documentation, tests aur
architecture cleanup ki zaroorat hai.


Achhi Cheezein
--------------
1. PDO database layer use ho raha hai.
   File: config/database.php
   Isse prepared statements aur safer DB access possible hai.

2. Authentication middleware present hai.
   File: includes/auth_check.php
   Login check, role check, session timeout aur login rate limiting jaise
   features already implemented hain.

3. CSRF helper available hai.
   File: includes/helpers.php
   Forms aur APIs ke liye token generation/verification ka base bana hua hai.

4. Migration runner available hai.
   File: database/migrate.php
   Database schema changes ko manage karne ke liye script already hai.

5. Feature coverage broad hai.
   Project mein wealth tracking ke kaafi modules hain, jo product potential ko
   strong banate hain.


Major Problems / Risks
----------------------
1. README almost empty hai.
   File: README.md
   Isse setup, migration, cron, testing aur project understanding difficult ho
   jaati hai.

2. Central API router bahut bada hai.
   File: api/router.php
   Yeh file approx 1379 lines ki hai aur bahut saare actions ek hi switch mein
   handle kar rahi hai. Isse debugging, security audit aur testing hard hoti hai.

3. CSRF exemption list risky lag rahi hai.
   File: api/router.php
   Kuch actions jo data change kar sakte hain, read-only exemption list mein
   dikh rahe hain. Add, delete, save, update, toggle, import, mark jaise actions
   par CSRF required hona chahiye.

4. Integration tests placeholder hain.
   File: tests/integration/test_apis.php
   Is file mein abhi real test cases nahi hain.

5. .env file project root mein present hai.
   File: .env
   Agar yeh git mein track ho rahi hai to secrets leak ka high risk ho sakta
   hai. .env ko ignore karna chahiye aur sirf .env.example commit hona chahiye.

6. .gitignore file visible nahi mili.
   Isse logs, cache, .env, vendor, backups jaise files accidentally commit ho
   sakte hain.

7. Architecture mixed hai.
   composer.json mein PSR-4 autoload WealthDash\\ => src/ defined hai, lekin
   current code mostly procedural folders mein hai. Future maintainability ke
   liye consistent architecture helpful hoga.

8. API error reporting suppress hai.
   File: api/router.php
   Production mein display errors off hona sahi hai, lekin proper structured
   logging ke bina debugging difficult ho sakti hai.

9. Database migrations ka structure scattered hai.
   SQL files database/ aur database/migrations/ dono jagah hain. Naming/order
   policy aur migration discipline improve karni chahiye.

10. Frontend JS bhi bada ho raha hai.
    File: public/js/app.js
    Is file mein approx 1627 lines hain. Feature-wise modules mein split karna
    better rahega.


High Priority Work
------------------
1. Security audit
   - CSRF exemption list audit karo.
   - Sirf pure read-only actions exempt rakho.
   - Add, edit, delete, save, update, toggle, import, mark actions par CSRF
     mandatory karo.
   - Admin actions ke liye extra confirmation, audit log aur rate limit lagao.

2. .env and secrets safety
   - .gitignore create/update karo.
   - .env ko ignore karo.
   - .env.example ko safe sample values ke saath rakho.
   - Agar .env kabhi commit hua hai to secrets rotate karo.

3. Router refactor
   - api/router.php ko domain-wise split karo:
     - api/routes/mutual_funds.php
     - api/routes/stocks.php
     - api/routes/fd.php
     - api/routes/goals.php
     - api/routes/admin.php
     - api/routes/reports.php
   - Action registry/map use kar sakte hain instead of giant switch.

4. Documentation
   README.md mein yeh sections add karo:
   - Project overview
   - Requirements
   - XAMPP setup
   - .env setup
   - Database migration command
   - Cron setup
   - Testing command
   - Main modules
   - Troubleshooting

5. Testing
   - API integration tests real banao.
   - Auth tests add karo.
   - CSRF required/not-required tests add karo.
   - Core finance calculators ke unit tests expand karo.
   - Migration dry-run test add karo.


Medium Priority Work
--------------------
1. Structured logging
   - API request id add karo.
   - Error logs mein action, user_id, route aur exception message record karo.
   - Sensitive data logs mein avoid karo.

2. Admin safety
   - DB truncate, restore, cron trigger jaise actions ke liye double-confirm
     flow banao.
   - Admin actions ka audit log mandatory karo.

3. Code organization
   - Procedural APIs ko gradually service classes mein move karo.
   - Shared validation helpers banao.
   - Repeated JSON/action patterns ko centralize karo.

4. Frontend cleanup
   - public/js/app.js ko smaller modules mein split karo.
   - Feature-specific JS files ko consistent naming aur loading pattern do.
   - Common API client, toast, modal aur validation helpers centralize karo.

5. Migration discipline
   - Migrations ka single folder policy decide karo.
   - Naming convention fix karo.
   - Rollback limitation document karo, kyunki current rollback SQL reverse
     nahi karta, sirf migration log remove karta hai.


Low Priority / Future Enhancements
----------------------------------
1. Static analysis
   - PHPStan ya Psalm add kar sakte hain.
   - PHP_CodeSniffer already config present lag raha hai, use enforce karo.

2. Composer scripts
   composer.json mein scripts add karo:
   - test
   - lint
   - migrate
   - migrate:dry-run

3. API versioning
   - Existing rest/v1 structure ko standardize karo.
   - Public API aur internal AJAX API ko clearly separate karo.

4. Performance
   - Heavy dashboard queries ke liye cache tagging already available hai.
   - Slow query logging aur query profiling add kar sakte hain.

5. Product polish
   - Dashboard empty states
   - Data import progress
   - Better error messages
   - Mobile layout audit
   - Accessibility improvements

1. .gitignore + .env safety
2. CSRF exemption audit
3. README setup guide
4. Basic integration tests

In 4 cheezon se project ka risk kam hoga aur future development faster hoga.
