// CLAUDE PROMPT — Admin Password Recovery (CLI Script)

Main DevVault Pro ka config.php de raha hoon (reference ke liye).

Kaam karna hai — ek CLI-only password reset script banao: reset_admin_password.php

Requirements:

1. Script SIRF command line se chale — agar koi browser se access kare to error de aur exit ho jaye:
   `if (php_sapi_name() !== 'cli') { die("This script can only be run from command line.\n"); }`

2. config.php require/include karo taaki DB connection mile (get_db() function use karo)

3. Interactive CLI prompts:
   - Sabse pehle saare users list karo (id, username, role) taaki pata chale kis user ka reset karna hai
   - Username/ID pucho kiska password reset karna hai
   - Naya password pucho (CLI input se — readline ya STDIN)
   - Confirm pucho (dobara type karwao, match check karo)
   - Minimum 8 characters validate karo

4. Password ko password_hash($new, PASSWORD_DEFAULT) se hash karo (jaisa admin.php mein hai — consistent rahe)

5. UPDATE users SET password_hash=?, password_changed=0 WHERE username=? 
   (password_changed=0 set karo taaki agla login par force password change ho — login.php mein yeh already check hota hai)

6. Success message print karo: "Password reset for user: [username]. They will be forced to change it on next login."

7. Script ke end mein ek clear warning print karo:
   "⚠️ IMPORTANT: Is script ko turant delete kar do ya .htaccess se block kar do agar yeh project folder mein webserver-accessible jagah par hai."

8. Script ke top mein comment likho:
   "RUN: php reset_admin_password.php  (command line se hi chalana — kabhi browser se access mat karna)"

IMPORTANT:
- Koi web output/HTML mat likho — pure CLI text output
- log_activity() call mat karo (kyunki yeh DB-direct operation hai, session nahi hoga) — bas ek simple text log file (logs/cli_reset.log) mein timestamp + username likh do
- Final complete reset_admin_password.php do
