Encryption fix karo — PBKDF2 key derivation add karo
CRITICAL config.php .env ⏱ 3 hrs
›

Problem

Abhi ENCRYPT_KEY directly AES-256 key ki jagah use ho rahi hai. PHP openssl_encrypt string ko truncate/pad karta hai jo proper 256-bit key nahi hai. Sahi tarika hai PBKDF2 se key derive karna. Saath mein default key bhi weak hai jo kabhi change nahi hoti.

⚠️ Yeh change karne ke baad purane encrypted passwords re-encrypt karne padhenge. Claude ko yeh migration bhi likhwao.

Files Deni Hain Claude Ko

config.php .env

Claude Ko Prompt Do

Mein DevVault Pro ka config.php aur .env de raha hoon. Encryption system fix karna hai.

Problems:
1. ENCRYPT_KEY directly openssl_encrypt mein pass ho rahi hai — yeh galat hai
2. Default key weak hai aur kabhi change nahi hoti
3. Key derivation (PBKDF2) nahi hai

Kaam karna hai:
1. `.env` mein `ENCRYPT_SALT` add karo (random 32-byte hex value generate karke)
2. `config.php` mein `derive_key()` function banao jo PBKDF2-SHA256 use kare (100,000 iterations)
3. `encrypt_val()` aur `decrypt_val()` functions update karo ki derive_key() use karen
4. Backward compatibility: purane encrypt kiye gaye values bhi decrypt ho saken (detect karke)
5. Ek migration script banao `migrate_reencrypt.php` jo existing DB ke saare encrypted fields purani method se decrypt kare aur nayi method se re-encrypt kare
6. Migration script mein progress show karo aur error handling rakho

IMPORTANT:
- Existing decrypt karne ki capacity maintain karo (old format detect karo)
- Final complete files do: config.php, .env (example), migrate_reencrypt.php
- Sabse pehle migrate_reencrypt.php chalana hoga, uske baad config.php replace karna
- Saara kaam test karke do — round trip encrypt → decrypt verify karo

Kya Karna Hai File Milne Par

  1. Pehle DB ka backup lo: data/vault.db copy karo
  2. migrate_reencrypt.php project mein rakh do
  3. Browser mein migrate_reencrypt.php kholo — migration chalao
  4. Migration complete hone par config.php replace karo
  5. .env mein ENCRYPT_SALT add karo
  6. App test karo — passwords decrypt ho rahe hain?
  7. migrate_reencrypt.php delete karo

HTTP Security Headers add karo
HIGH auth.php ⏱ 30 min
›

Problem

Koi bhi security headers nahi hain: no CSP, no X-Frame-Options, no X-Content-Type-Options. App ko iframe mein embed kiya ja sakta hai (clickjacking), aur browser MIME sniffing attacks possible hain.

Files Deni Hain Claude Ko

auth.php

Claude Ko Prompt Do

Mein DevVault Pro ka auth.php de raha hoon.

Kaam karna hai — yeh HTTP security headers add karo auth.php mein, session_start() ke baad aur kisi bhi output se pehle:

1. X-Frame-Options: DENY (clickjacking protection)
2. X-Content-Type-Options: nosniff (MIME sniffing protection)
3. Referrer-Policy: same-origin (URL leak prevention)
4. Content-Security-Policy — app mein inline CSS aur JS hain, Google Fonts bhi hain, to:
   - default-src 'self'
   - style-src 'self' 'unsafe-inline' https://fonts.googleapis.com https://cdn.jsdelivr.net
   - font-src https://fonts.gstatic.com
   - script-src 'self' 'unsafe-inline'
   - img-src 'self' data:
5. X-XSS-Protection: 1; mode=block (legacy browsers ke liye)

IMPORTANT:
- Headers sirf ek baar send hone chahiye (headers already sent error avoid karo)
- login.php ko exclude mat karo — login page par bhi headers chahiye
- Final complete auth.php do jo main replace kar sakun
