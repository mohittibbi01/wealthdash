T-01 — Security Fixes


PROJECT: DevVault Pro v2.5 (PHP + SQLite internal LAN tool)
TASK: T-01 — Security fixes (3 issues)
Existing codebase structure:
- config.php — DB connection, encryption functions, schema, ENCRYPT_KEY hardcoded here
- auth.php — session/login handling
- login.php — login form
- admin.php — user management
- api.php — AJAX endpoints

Fix these 3 issues:

FIX 1 — Move ENCRYPT_KEY to .env file
- Create a .env file in project root: ENCRYPT_KEY=DevVaultPro_S3cur3_K3y_2025_Ch4ng3Me!
- In config.php, read it using: $lines = file('.env'); parse manually (no external library)
- .env should be listed in README as "do not share / do not commit"
- All existing encrypted data must still decrypt correctly (same key, just different source)

FIX 2 — File upload MIME type validation
- Current code in config.php safe_upload() only checks pathinfo extension
- Add finfo_file() check to verify actual MIME type matches allowed list
- Allowed map: pdf→application/pdf, jpg/jpeg→image/jpeg, png→image/png,
doc→application/msword, docx→application/vnd.openxmlformats-officedocument.wordprocessingml.document,
xls→application/vnd.ms-excel, xlsx→application/vnd.openxmlformats-officedocument.spreadsheetml.sheet,
zip→application/zip, txt→text/plain, csv→text/csv
- If MIME mismatch: reject with clear error message

FIX 3 — Force password change on first login for default admin
- Add column: password_changed INTEGER DEFAULT 0 in users table (migration safe)
- Default seeded admin (username=admin) should have password_changed=0
- After successful login, if password_changed=0 → redirect to change_password.php
- change_password.php on success → set password_changed=1
- Block all other pages if password_changed=0 (check in auth.php require_login)

Provide: updated config.php (relevant sections), updated auth.php, updated safe_upload(),
.env file content, and migration SQL if needed. Keep all existing functionality intact.

PROJECT: DevVault Pro v2.5 — T-03: Session timeout with countdown timer

REQUIREMENT:
- Idle timeout: 5 minutes (300 seconds)
- Any user activity (mousemove, keypress, click, scroll) resets timer to 300s

with a live countdown and an "Extend Session" button
- If user clicks Extend: AJAX call to session_keep_alive endpoint → timer resets to 300s
- If user does NOT click within 60s: auto-redirect to logout.php
- Timer display: small unobtrusive element on topbar (e.g. "Session: 4:32") always visible

- Pure vanilla JS (no libraries)

returns {ok:true}
- Timer JS should be in a separate file: session_timer.js, included in all pages via
a common header include
- On server side: PHP session gc_maxlifetime should match (set in config.php:
ini_set('session.gc_maxlifetime', 360) — 1 min buffer)

Provide: session_timer.js (complete), api.php keepalive action addition,
config.php session settings, and HTML snippet to include timer display in topbar.

T-05 — Visitor Count History


PROJECT: DevVault Pro v2.5 — T-05: Visitor count time-series history

CREATE TABLE project_visitor_log (

site_last_update_date DATE,

entered_at DATETIME DEFAULT CURRENT_TIMESTAMP,
remarks TEXT,
FOREIGN KEY(project_id) REFERENCES projects(id) ON DELETE CASCADE
);

- For each project that has existing total_visitor_counter: insert one row into

site_last_update_date = existing last_update_date
- Keep original columns in projects table (don't drop them) but auto-sync on new entries

UI CHANGES:
- Project form: replace single visitor count field with "Add Visitor Log Entry" section:
entry_date (date picker, default today, cannot be future), visitor_count (number),
site_last_update_date (date), remarks
- Show last 5 entries in a small table on project detail
- Button to view full log history

REPORT (in report.php or new visitor_report.php):
- Select project (or all projects)
- Select date range: from_date to to_date (to_date cannot be future)
- Logic:
baseline = latest entry WHERE entry_date <= from_date (if none, use first entry ever)
final = latest entry WHERE entry_date <= to_date
net_gain = final.visitor_count - baseline.visitor_count
updates_count = COUNT(DISTINCT site_last_update_date) WHERE entry_date > baseline.entry_date
AND entry_date <= to_date AND site_last_update_date != baseline.site_last_update_date
- Output table: Project Name | Baseline Count | Baseline Date | Final Count | Final Date |
Net Gain | Site Updates in Period
- CSV export

T-09 — Global Task / Work Order Module


PROJECT: DevVault Pro v2.5 — T-09: Global task / work order module
(Assumes T-04 done — tech_subtype available. T-07, T-08 done for reference on module pattern.)

THIS IS THE MOST COMPLEX MODULE — build carefully.

SCHEMA:
CREATE TABLE work_orders (
id INTEGER PRIMARY KEY AUTOINCREMENT,
title TEXT NOT NULL,
description TEXT,
instruction_source TEXT, -- e.g. verbal, written, order number
applicable_tech TEXT, -- JSON array: ["NET","AEM","WebMyWay"] or ["ALL"]
scope TEXT DEFAULT 'all', -- 'all' = all sites of selected tech, 'specific' = selected list
priority TEXT DEFAULT 'Normal', -- High / Normal / Low
deadline DATE,
assigned_to TEXT,
created_by INTEGER,
status TEXT DEFAULT 'Active', -- Active / Completed / Cancelled
created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE work_order_sites (
id INTEGER PRIMARY KEY AUTOINCREMENT,
work_order_id INTEGER NOT NULL,
project_id INTEGER NOT NULL,
site_status TEXT DEFAULT 'Pending', -- Pending / Done
done_by INTEGER,
done_at DATETIME,
remarks TEXT,
UNIQUE(work_order_id, project_id),
FOREIGN KEY(work_order_id) REFERENCES work_orders(id) ON DELETE CASCADE,
FOREIGN KEY(project_id) REFERENCES projects(id) ON DELETE CASCADE
);

CREATE TABLE work_order_history (
id INTEGER PRIMARY KEY AUTOINCREMENT,
work_order_id INTEGER NOT NULL,
project_id INTEGER, -- NULL if overall order status changed
action TEXT,
changed_by INTEGER,
changed_at DATETIME DEFAULT CURRENT_TIMESTAMP,
detail TEXT
);

LOGIC — when work order is created:
- If applicable_tech = ["ALL"] and scope = "all": auto-populate work_order_sites with
ALL active projects
- If applicable_tech = ["WebMyWay"] and scope = "all": auto-populate with all WebMyWay projects
- If scope = "specific": admin manually selects which projects from a checklist

BULK COMPLETE LOGIC:
- On work order detail page: show progress bar (X of Y sites done)
- Per technology group, show "Mark All [WebMyWay] Done" button
- This one button marks ALL Pending sites of that tech as Done in one DB operation
- For .NET / AEM: each site has individual "Mark Done" button only (no bulk for these)
- Log every bulk action in work_order_history

UI — workorders.php:
- List all work orders with: Title | Tech Scope | Priority | Deadline | Progress (X/Y) | Status
- Work order detail:
Top: overview + bulk complete buttons per tech
Below: site-by-site table with individual done/undo buttons
Bottom: activity history log
- Add work order form: title, description, source, tech multi-select, scope radio,
priority, deadline, assign to

Provide: complete schema, workorders.php (full file), api.php bulk_complete action,
and progress calculation query.
