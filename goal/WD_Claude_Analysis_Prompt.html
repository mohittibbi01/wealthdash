Main tumhe apna WealthDash project de raha hoon — ek personal finance dashboard (PHP 8.1 backend + vanilla JS frontend, MySQL 8). Tumhe is project ko deeply analyze karna hai aur ek standalone HTML file banana hai. --- TASK: Ek file banana hai: wealthdash_bugs_v[YYYYMMDD].html Yeh file: 1. SPRINT PLAN banaye (4 weeks) 5. UI bilkul wealthdash_master_v54.html jaisi ho (same dark theme) --- ANALYSIS STEPS — In order mein karo: STEP 1 — zzProjectList.txt padhna: Poori file tree dekho. Note karo file sizes. Under 600 bytes = stub/empty file. STEP 2 — api/router.php audit: Har api/ subfolder ke liye check karo — router.php mein us folder ka route hai ya nahi. Dead routes (route hai, file nahi) aur Missing routes (file hai, route nahi) — dono note karo. STEP 3 — Stub files identify karo: Har PHP file jo 600 bytes se kam hai — stub hai. List banao. Known stubs: api/ai/fund_recommend.php, api/ai/goal_advisor.php, api/ai/news_impact.php, api/ai/sip_optimizer.php, api/automation/rules.php, api/broker/zerodha_sync.php STEP 4 — includes/ audit: error_handler.php exist karta hai? validator.php? cache.php? response_helper.php? Agar nahi hain toh MISSING mark karo. STEP 5 — templates/ vs api/ cross-check: Template hai lekin API nahi → missing API. API hai lekin template nahi → missing UI. STEP 6 — migrations/ audit: Latest migration number kya hai? Koi indexes missing hain? STEP 7 — public/js/ audit: Duplicate functions? Dead code? Missing implementations? STEP 8 — Security check: Rate limiting sab endpoints pe? Input validation centralized? CSRF protection? STEP 9 — wealthdash_master_v54.html se pending tasks: Tasks jo done:true nahi hain — list banao. Actual code se verify karo. --- OUTPUT FILE FORMAT: Dark theme CSS variables (exact): --bg: #060c1a --surface: #0d1528 --surface2: #141f35 --surface3: #1b2840 --border: #1f2e4a --text: #d6e4ff --muted: #5a7aaa --accent: #4fa3ff --accent2: #38e8b0 --p0: #ff4040 (critical) --p1: #ff8c42 (high) --p2: #f0b429 (medium) --p3: #5a7aaa (low) File structure: - Sticky header: WealthDash logo + total stats (X bugs, Y tasks, Z improvements) - Tab bar: [Bugs] [Pending Tasks] [New Improvements] [Sprint Plan] - Filter bar: All | P0 | P1 | P2 | P3 — aur Worker: W1 | W2 | W3 | W4 | W5 | M - Search box: title/id se search BUG CARD format (per bug): - ID: BUG-001 - Priority: P0/P1/P2/P3 - Type: syntax / missing / logic / perf / security / ux - Title - File(s) affected - Impact - Root cause - Recommended fix PENDING TASK format (per task): - ID: from master file (t001, tnew01, etc.) - Title - Priority + Worker - Estimated effort - Files needed - Why pending (evidence: stub file / no route / no template) NEW IMPROVEMENT format: - IMPR-001 style ID - Title - Type: architecture / feature / security / dx / testing - Impact: High/Medium/Low - Effort: High/Medium/Low - Detail SPRINT PLAN: 4 weeks, P0 first, group related tasks. --- IMPORTANT RULES: 1.
