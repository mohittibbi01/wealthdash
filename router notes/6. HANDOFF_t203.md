## API Actions
| Action | Method | Params |
|--------|--------|--------|
| `ppf_deposits_list` | GET | — |
| `ppf_deposit_get` | GET/POST | scheme_id, [fy_year] |
| `ppf_deposit_history` | GET/POST | scheme_id |
| `ppf_deposit_save` | POST | scheme_id, amount, date, [note], [override=0] |
| `ppf_deposit_delete` | POST | scheme_id, fy_year, entry_id |

## Logic
- `override=1`: sets total directly (for bulk/manual entry — existing post_office.php behaviour)
- `override=0` (default): appends entry, validates cumulative ≤ ₹1,50,000, rejects over-limit
- FY computed from deposit date (Apr–Mar cycle)
- Entries stored as JSON array `[{id, date, amount, note}]` for per-entry delete
